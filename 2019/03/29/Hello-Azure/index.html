    Hello Azure

    adingx's note
    2019-03-29
    cloud

    In February of 2010, Azure Platform is available for commercial use. Later 3 years than Amazon Web Service. Initially it pay more focus on big company customer from Windows Server user. And then it gradually open to all users. Until 2018, AWS remains the global market share leader in public cloud services at 33% followed by Azure at 13% and Google Cloud at 6%, from Synergy Research Group Report. Yet I prefer to Azure for friendly and amiable UEX.

    Quick Start

    At first, open azure-console to register your azure account and a credit card binding is required. After sign in, so many items and options are boomed to your screen. From my beginning, start with Azure VM(IaaS) or Azure Function(FaaS) is good choice.

    fig-1: IaaS,FaaS,SaaS,PaaS

    FaaS is similar to PaaS.
    The main difference that FaaS is unified event-driven API to support all programming language, and PaaS is depended on Middleware interface.

    Azure Virtual Machine

    Create a Virtual Machine in azure-portal, selecting CPU/Memory/HD Storage similar to setup a VM on desktop.
    OS setup is so easy on Azure Cloud, ubuntu or debian.
    Pay attention to user auth for the way how to ssh connect your VM on Azure cloud, user password or certificates.

    • Create a Virtual Machine
    • Select the auth strategy
    • Networking setting
    • SSH connect and deploy your code
